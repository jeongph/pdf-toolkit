# 안전 원칙

PDF 수정은 파괴적 작업이다. 다음 원칙을 모든 작업에서 준수한다.

## 원본 보존이 기본

| 상황 | 동작 |
|---|---|
| `-o`로 출력 경로 명시 | 그 경로로 출력. 원본 그대로 |
| 출력 미명시, `--in-place` 없음 | `<원본>.modified.pdf` 또는 `.reordered.pdf` 등으로 자동 생성 |
| 출력 미명시, `--in-place` 명시 | 원본 덮어쓰기 (incremental save 사용) |
| 출력==입력, `--in-place` 없음 | 작업 거부 (실수 방지) |

## 파괴적 작업 사전 확인

다음 경우는 `AskUserQuestion`으로 한 번 더 확인:

- 5페이지 이상 삭제
- 전체 페이지 순서 대규모 변경 (절반 이상 위치 이동)
- `--in-place` 사용 시 모든 작업
- 출력 파일이 이미 존재하고 덮어쓰기되는 경우 (`-o`로 기존 파일 지정 시)

## 검증 후 실행

작업 전 반드시 검증:

1. **입력 PDF 존재 여부** — `os.path.exists`로 확인
2. **페이지 인자가 PDF 페이지 범위 내** — `parse_pages()`가 자동 필터링
3. **순서 변경 시 모든 페이지 참조** — 누락·중복 검사
4. **병합 시 입력 PDF 두 개 이상** — 단일 PDF 병합은 무의미

## 비파괴 변환 우선

가능하면 메타 변경만으로 처리:
- 회전 → `page.set_rotation()` (페이지 내용 변경 없음)
- 메타데이터 → `doc.set_metadata()` (이미지·텍스트 그대로)

페이지 콘텐츠 자체를 변형해야 하는 작업(크롭, 압축 재인코딩 등)은 v0.2+에서 도입.

## 작업 후 검증 권장

수정 결과를 `pdf-scan-audit`으로 한 번 더 검사하면 의도치 않은 변형(페이지 잘림, 메타 손상 등)을 조기에 발견할 수 있다.

```
/pdf-rotate book.pdf 54 180 -o fixed.pdf
/audit-pdf fixed.pdf
```
